# 夜市麻將記憶賓果 — 核心玩法

> 一句話：**記住每張牌原本的位置 → 追蹤洗牌軌跡 → 翻它讓它回家**

---

## 1｜核心概念

桌上 6×6 共 36 張**獨特**的麻將牌。

- Phase 1 看到的初始配置 = **這局的賓果目標**
- Phase 2 洗牌只是把「位置打散」，牌本身（一萬／東／發⋯）沒變
- Phase 3 玩家翻洗牌區某張牌 → 那張牌「**飛回它本來該在的位置**」（在右邊 Bingo 區）
- Bingo 區慢慢補實 → 完整一條橫／直／主對角 = Bingo

> 沒有獨立的賓果卡。**初始位置就是賓果板。**

---

## 2｜與傳統夜市麻將的差異

| 項目 | 傳統夜市麻將賓果 | 本作 |
|---|---|---|
| 賓果板 | 獨立卡片（樂透卡） | 即初始 6×6 配置 |
| 翻牌後 | 拿在手上對照賓果卡 | 自動飛回原位 |
| 連線判定方式 | 對照卡片標記 | 補實連線格 |

## 3｜麻將牌組成（36 張獨特）

| 類別 | 內容 | 張數 |
|---|---|---|
| 萬 | 一萬 ~ 九萬 | 9 |
| 筒 | 一筒 ~ 九筒 | 9 |
| 條 | 一條 ~ 九條 | 9 |
| 風 | 東 南 西 北 | 4 |
| 字 | 中 發 白 | 3 |
| 花 | 梅 蘭 | 2 |
| **合計** | | **36** |

> 不像傳統麻將每張 4 張，這裡**每張只用 1 張**。

---

## 4｜四個 Phase 詳細流程

### Phase 1 — 展牌記憶（10 秒）
- 36 張牌全部牌面朝上排成 6×6
- 玩家可自由觀看
- 最後 3 秒倒數變紅 + 提示音

### Phase 2 — 蓋牌洗牌
1. 所有牌翻面蓋牌
2. 攤主開始洗牌（位置交換動畫）
3. 玩家可全程追蹤每張牌的軌跡
4. 洗牌完成 → 進入 Phase 3

### Phase 3 — 玩家翻牌（最多 15 次）
- 玩家從洗牌區自由選位置翻牌
- 每翻開一張：
  - 顯示牌面（如「東」）
  - 那張牌**飛回 Bingo 區的目標位置**（= 它在 Phase 1 的位置）
  - 洗牌區該格變空
  - **如果這張牌補滿某條線 → 該線即時發光得分**
- 玩家總共 **15 次翻牌機會**

### Phase 4 — 結算
- 顯示完成線數、評價、獎勵
- 完成連線可兌換娃娃

## 5｜規則細項（已確認）

| # | 規則 |
|---|---|
| ① | **每局 36 張牌會隨機到不同位置**（每局 Phase 1 的初始 6×6 都不同） |
| ② | **連線即時發光**（補實當下就算 Bingo，不用等結算） |
| ③ | **不能翻空格**（已翻過的位置點擊無效，不消耗次數） |
| ④ | **只有真的翻成功一張才算一次**（誤觸／空格／取消都不算） |
| ⑤ | **只算 2 條主對角線**（左上→右下、右上→左下） |

---

## 6｜連線判定（可能的 14 條）

| 類型 | 條數 | 說明 |
|---|---|---|
| 橫線 | 6 | 第 0~5 列 |
| 直線 | 6 | 第 0~5 行 |
| 主對角 | 2 | (0,0)-(5,5)、(0,5)-(5,0) |
| **合計可能** | **14** | |

**評價上限為 3 條**（見下表）。

### 為什麼最多就是 3 條？
- 1 條線需要 6 張牌
- 2 條交叉線共用 1 張，需要 6+6-1 = **11 張**
- 3 條全相交共用 3 張，需要 6+6+6-3 = **15 張**（剛好等於翻牌次數）
- 所以 15 次翻牌的物理上限 = 3 條，難度 3 條 = 完美 run

## 8｜核心策略（玩家視角）

玩家在腦中要同時做三件事：

1. **記憶**（Phase 1）：記下「東在 (0,0)、七萬在 (2,3)、…」
2. **追蹤**（Phase 2）：盯住關鍵牌被洗到哪裡
3. **取捨**（Phase 3）：集中補某一條線，比亂翻更划算

**難度來自「資訊衰減」**：洗牌越快／越多步驟 → 玩家追蹤越容易斷線 → 只能靠記憶 + 賭。

## 9｜資料結構（給工程參考）

```js
GameState = {
  // 36 格的 Bingo 目標配置（= Phase 1 初始排列）
  // index 0~35 對應 6×6 (row * 6 + col)，值是 tile.id
  bingoTarget: number[36],

  // 洗牌區當前狀態
  // 值是 tile.id 或 null（已翻空）
  shuffleArea: (number | null)[36],

  // Bingo 區當前狀態
  // true = 已補實
  bingoRevealed: boolean[36],

  // 剩餘翻牌次數
  flipsRemaining: number,    // 起始 15

  // 已完成連線
  completedLines: Array<{ type: 'row'|'col'|'diag', index: number }>,
}
```

**翻牌動作偽碼**：
```
flipTile(state, shuffleIndex):
  tile = state.shuffleArea[shuffleIndex]
  if tile === null: return ERROR_EMPTY  // 規則 ③，不扣次數

  targetIndex = state.bingoTarget.indexOf(tile)
  state.shuffleArea[shuffleIndex] = null
  state.bingoRevealed[targetIndex] = true
  state.flipsRemaining -= 1                // 規則 ④

  newLines = detectNewLines(state)
  if newLines.length > 0: trigger 發光動畫    // 規則 ②
  state.completedLines += newLines
```
